00029 //---------------------------------------------------------------------------
00030 //
00031 // Class Synopsis: class Prbs_gen
00032 //
00033 // Description:
00034 //   Pseudo random binary sequence generator using polynomial
00035 //   division in GF (2).
00036 //
00037 // There are two ways to built such a generator. Both use some
00038 // form of shift register.
00039 //
00040 // 1. The first type feeds back the parity (XOR) of the taps corresponding to
00041 //    the non-zero elements of the polynomial into the input of the register.
00042 //    This is the most efficient way to do it in hardware. 
00043 //
00044 // 2. In the seond form, when the bit shifted out is 1, the contents of the
00045 //    register are XORed with a bit pattern representing the polynomial.
00046 //    This is the best way to do it in software.
00047 //
00048 // Mutatis mutandis the two forms are equivalent. Any sequence that can be
00049 // generated by one of the realisations can also be produced by the other.
00050 // This software obviously uses the second form. It can use any polynomial
00051 // up to (and including) a degree of 32.
00052 //  
00053 //
00054 //  set_poly (p)
00055 //
00056 //    Defines the polynomial to be used. The value of p is found from the
00057 //    sequence of coefficients (0 or 1) of the polynomial starting with the  
00058 //    constant term, and dropping the highest one
00059 //
00060 //                                   0 1 2 3 4 5 6 7
00061 //    Example: P = x^7 + x^6 + 1 --> 1 0 0 0 0 0 1 1 --> 1000001 --> 0x41
00062 //
00063 //    To emulate the first form described above, start with the highest
00064 //    exponent and drop the constant term.
00065 //
00066 //                                   7 6 5 4 3 2 1 0
00067 //    Example: P = x^7 + x^6 + 1 --> 1 1 0 0 0 0 0 1 --> 1100000 --> 0x60
00068 //
00069 //    Also sets the state to all ones.
00070 //
00071 //   
00072 //  set_state (x)
00073 //   
00074 //    Sets the initial state to x.
00075 // 
00076 //
00077 //  step ()
00078 //
00079 //     Returns the next pseudo random bit.
00080 //
00081 //
00082 //  sync_forw (x)
00083 // 
00084 //    This sets the generator in a state as if the last N (= degree) bits 
00085 //    were those defined by x (the LSB of x represents the oldest bit).
00086 //    This can be used to synchronise a BER counter to a received bit stream,
00087 //    or to set the initial state when emulating a generator of the first form
00088 //    when the output is taken from the feedback.
00089 //
00090 //
00091 //  sync_back (x)
00092 //
00093 //    This sets the generator in a state so that the first N (= degree) output
00094 //    bits will be those defined by x (the LSB of x will be the first output bit).
00095 //    This can be used to set the initial state when emulating a generator of
00096 //    the first form when the output is taken from the shifted out bit.
00097 //
00098 //
00099 //---------------------------------------------------------------------------==
